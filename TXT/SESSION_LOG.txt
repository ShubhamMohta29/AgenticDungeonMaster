  BUILT
  ─────

  1. Dark-mode-only refactor
     — Removed ThemeProvider and ThemeToggle entirely.
     — Hardcoded `class="dark"` on <html> in app/layout.tsx.
     — Removed components/providers/ directory.
     — Rationale: light mode broke the dark-fantasy atmosphere. Theme toggle was
       wasted complexity. "Dark mode only" is now a documented design rule.

  2. Play page 3-column layout rebuild (app/(game)/campaign/[id]/play/page.tsx)
     — Left column (w-64): InitiativeTracker stacked above DiceLog.
     — Center column (flex-1): StoryLog fills available height, ActionPanel
       pinned below. ActionPanel hidden for archived campaigns.
     — Right column: CharacterPanel (fixed width from component).
     — Top bar (h-14): campaign name (amber italic), current_scene (truncated,
       hidden on small screens), ← Dashboard link.
     — Archived banner below top bar when campaign.status === 'completed'.
     — `isArchived` flag gates ActionPanel, shows read-only badge.

  3. ManaBar component (components/ui/ManaBar.tsx)
     — Sums character.spells.slots and character.spells.slot_max.
     — Returns null for NON_SPELLCASTERS: fighter, barbarian, monk, rogue.
     — Blue gradient bar. Wired into CharacterPanel below the HP bar.

  4. "Jump to latest" in StoryLog (components/game/StoryLog.tsx)
     — Amber button appears when user has scrolled >200px above the bottom.
     — Clicking it smooth-scrolls to the bottom of the message list.

  5. ActionPanel error recovery (components/game/ActionPanel.tsx)
     — Added dmError?: string | null and onClearError?: () => void props.
     — Inline amber-bordered banner shows error text + "Try again" / "×" buttons.
     — play/page.tsx owns dmError state, passes it down, clears on new action.

  6. Campaign end / archive flow
     — app/api/campaign/end/route.ts:
         Auth + membership check. Fetches campaign, characters, last 10
         narration messages, world memory, quests. Calls Groq Llama 3.3 70B
         with epic epilogue prompt (maxTokens: 600). Returns { summary }.
     — app/api/campaign/archive/route.ts:
         Auth + membership check. Sets campaigns.status = 'completed'.
     — components/game/CampaignEndModal.tsx:
         Two-step: loads AI summary on mount (useEffect), shows it, then
         "End Campaign" confirm button calls archive route, then
         router.push('/dashboard').
     — play/page.tsx: END_CAMPAIGN_RE regex detects "end/finish/conclude/
       close/archive ... campaign" intent → opens CampaignEndModal.

  7. Rate limiter (lib/rateLimiter.ts)
     — In-memory Map<string, { lastCall, count }>.
     — checkRateLimit(key, { windowMs, max }) returns { allowed, retryAfter }.
     — Wired into /api/dm/route.ts: 1 DM call per 3 seconds per campaign.
     — Returns HTTP 429 with retryAfter on violation.

  KNOWN BUGS / OPEN ITEMS
  ────────────────────────

  [CRITICAL — UNRESOLVED]
  After character creation, the play page opens and immediately redirects
  to /dashboard. Root cause: loadData in play/page.tsx runs a campaign
  fetch. Supabase RLS checks campaign_members to authorize the read.
  The newly-created character row triggers the redirect to /play before
  the campaign_members INSERT has fully propagated through RLS. So the
  campaign query returns null → router.push('/dashboard').

  Fix: Retry the campaign fetch up to 3× with ~600ms delay before hard-
  redirecting. See PENDING WORK below.
